# SEO Audit — weightlossrankings.org

**Date:** 2026-04-06
**Auditor:** Automated crawl + manual review
**Scope:** 8 sample pages, sitemap, robots.txt, keyword & content gap analysis
**Method:** Raw HTML fetch via curl (User-Agent: Chrome desktop), regex extraction of meta, heading, schema, link, and asset data.

---

## 1. Executive Summary

### Top 5 Wins (things the site already does well)

1. **Clean technical foundation.** HTTPS enforced, HSTS set, 307 apex → www redirect works, single H1 on every audited page, viewport meta present, preload/preconnect hints in place, only 1 stylesheet + ~8 scripts (lean). No inline style blocks.
2. **Sitemap is comprehensive and fresh.** 183 URLs, logical priority weighting (home=1.0, category=0.9, review=0.8, state=0.8), daily lastmod on dynamic pages, correctly declared in robots.txt.
3. **Schema markup is implemented meaningfully on money pages.** `/best/semaglutide-providers` ships `ItemList` + `FAQPage`; `/reviews/coreage-rx` ships `Product` + `Review` + `AggregateRating`; `/drugs/semaglutide` ships `Drug` + `FAQPage`. These are the three schema types Google rewards most in this niche.
4. **Title tags are keyword-led and mostly within length budget.** Every audited page has a unique, descriptive `<title>` (61–80 chars) — no duplication, no generic "Home" titles.
5. **Programmatic coverage is strong.** The site already has 50 state pages, 6 drug pages, 20+ reviews, 60+ comparison pages, and /alternatives/* pages — the hard part (scale) is done. Most remaining wins are template-level edits that propagate across hundreds of URLs.

### Top 5 Issues (in priority order)

1. **CRITICAL — No `<link rel="canonical">` on ANY audited page.** 0/8 pages declared a canonical. With the apex-to-www 307 redirect, parameterized compare URLs, and programmatic pages, this is a duplicate-content time bomb. Google will pick canonicals itself, often wrongly.
2. **CRITICAL — Open Graph tags are identical site-wide.** Every page returns the same `og:title`, `og:description`, and `og:image` (the homepage values). Social shares of a review, drug page, or comparison all look like the homepage. Massive lost CTR on Twitter/Slack/LinkedIn/iMessage link previews.
3. **HIGH — Zero `<img>` tags on every audited page.** Pages are text-only. No hero images, no provider logos as images, no chart screenshots, no product photos. This hurts image search traffic, reduces dwell time, and makes pages feel thin. (It also means "alt text coverage" is trivially 100% because the denominator is zero.)
4. **HIGH — Thin content on commercial intent pages.** `/compare` = 268 words, `/savings-calculator` = 197 words, `/compare/coreage-rx-vs-hims` = 465 words, `/reviews/coreage-rx` = 499 words. These are commercial/transactional queries where top-ranking competitors ship 1,500–3,500 words. Review and comparison pages in particular need 2–3x more depth.
5. **HIGH — Missing BreadcrumbList schema on 7/8 pages** (only compare-cvh has it), and missing meta description on nothing but several are too short (`/reviews/coreage-rx` = 120 chars, `/states/texas` = 138 chars). Also **no `<h2>` on `/compare`** — the page jumps H1 → H3 (footer sections), breaking heading hierarchy.
